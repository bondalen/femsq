# Правила работы с отчётами JasperReports (FEMSQ)

1. Строгое соответствие исходнику: тексты, координаты (twips→points), размеры, цвета (forecolor/backcolor), выравнивание, форматы (pattern) и логика должны повторять экспорт Access/источник. Любые отклонения — только по согласованию и с фиксацией в документации.
2. Данные и вычисления: поля, выражения (IIf → Java ternary), условия отображения и группировки должны совпадать с исходным отчётом; скрытые элементы (Visible=False) не добавлять, если явно не согласовано.
3. Цвета и шрифты: использовать цвета и шрифты из исходника; для условного форматирования (VBA) применять conditional styles/printWhenExpression с теми же цветами и порогами.
4. Геометрия: сохранять координаты/размеры по файлу экспорта; вертикальные уровни (y/height) оставлять как в исходнике; линии-разделители и баннеры — с исходными цветами и высотами.
5. Тестирование: перед сдачей — Preview в Jaspersoft Studio на тестовых данных; сверить визуально с исходным отчётом (приоритет — строгая идентичность).
6. Документация: любые изменения фиксировать в планах/журналах; если отступаем от исходника — обязателен комментарий в документе и согласование.
