# Предложение по обновлению Spring Framework/Boot

## Текущая ситуация

### Версии в проекте
- **Spring Boot**: `3.3.5` (в `femsq-web/pom.xml`)
- **Документация**: указывает `Spring Boot 3.4.5` (несоответствие!)
- **Java**: 21 LTS
- **Spring Framework**: определяется через Spring Boot (неявно)

### Предложение Cursor
Cursor предлагает обновить "Spring Framework 6.1.14" до "3.5", что выглядит **ошибочно**:
- Spring Framework 6.1.14 - это очень новая версия (если существует)
- Обновление до 3.5 - это откат, а не обновление
- Вероятно, Cursor путает Spring Framework и Spring Boot

## Анализ

### Spring Boot версии
- **Текущая**: 3.3.5 (используется в проекте)
- **Последняя стабильная**: 3.4.x (по состоянию на 2025)
- **Spring Boot 3.5**: пока не существует (следующая мажорная версия будет 4.0)

### Spring Framework версии
- Spring Framework 6.x используется в Spring Boot 3.x
- Spring Framework 6.1.14 - возможно, это версия, встроенная в Spring Boot 3.3.5
- EOL (End of Life) в 2026-06 - это нормально для Spring Boot 3.3.x

## Рекомендации

### Вариант 1: Обновить до Spring Boot 3.4.x (рекомендуется)

**Преимущества:**
- Актуальная стабильная версия
- Исправления безопасности
- Новые возможности
- Поддержка до 2027+

**Действия:**
1. Обновить `spring.boot.version` в `femsq-web/pom.xml` с `3.3.5` до `3.4.5`
2. Проверить совместимость зависимостей
3. Запустить тесты
4. Обновить документацию

**Риски:**
- Минимальные (минорное обновление)
- Возможны небольшие breaking changes в зависимостях

### Вариант 2: Оставить текущую версию 3.3.5

**Преимущества:**
- Стабильность
- Нет риска breaking changes
- Поддержка до 2026-06 (еще есть время)

**Недостатки:**
- Устаревание через год
- Отсутствие новых возможностей
- Возможные уязвимости безопасности

### Вариант 3: Игнорировать предложение Cursor

**Причина:**
- Предложение выглядит ошибочным (обновление до несуществующей версии 3.5)
- Cursor, вероятно, путает Spring Framework и Spring Boot

## План действий (рекомендуемый)

### Шаг 1: Исправить несоответствие в документации
- Обновить `project-docs.json`: `Spring Boot 3.4.5` → `Spring Boot 3.3.5`

### Шаг 2: Решить об обновлении
- **Рекомендация**: Обновить до Spring Boot 3.4.5
- **Альтернатива**: Оставить 3.3.5 до следующего релиза

### Шаг 3: Если обновляем до 3.4.5
```xml
<!-- В femsq-web/pom.xml -->
<properties>
    <spring.boot.version>3.4.5</spring.boot.version>
</properties>
```

### Шаг 4: Проверка после обновления
1. `mvn clean compile` - проверка компиляции
2. `mvn test` - запуск unit-тестов
3. `mvn verify -Pintegration` - запуск integration-тестов
4. Проверка работы приложения
