- `POST /{id}/execute` должен возвращать `202 Accepted` немедленно, не дожидаясь завершения ревизии.
- Если ревизия в статусе `RUNNING` — `POST /execute` возвращает `409 Conflict` с пояснением.
- Причина: сейчас `executeAudit` выполняется синхронно на HTTP-потоке; для заглушек (~900 мс) это незаметно, но с реальными Excel-файлами приведёт к таймауту HTTP и плохому UX.

##### Результат (исполнено)

- В `FemsqWebApplication` добавлен `@EnableAsync`.
- В `AuditExecutionServiceImpl.executeAudit` добавлен `@Async`, поэтому выполнение ревизии переносится в фон и POST возвращает `202 Accepted` без ожидания завершения.

### 7.3. Polling с условием остановки на фронтенде ✅

### 7.4. Снижение уровня логирования `ConnectionFactory` ✅

**Задачи:**
