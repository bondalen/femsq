# План работы: Подготовка к разработке импорта из Excel на основе действующей Access‑системы (nb-win, Office 2013)

**Дата создания:** 2026-03-10  
**Проект:** FEMSQ — подготовка к импорту исходных данных из листов Excel для ревизий  
**Цель:** Запустить действующие файлы Access/Excel на nb-win с MS Office 2013, подключить Access к тестовой БД MS SQL Server в Docker и, опираясь на работающую систему, формализовать сценарий «ревизия → импорт Excel», структуру временных таблиц и архитектуру импорта в FEMSQ.

---

## 🧩 Этап 0: Подготовка среды (Office 2013 + Access/Excel на nb-win)

### 0.1 Установка MS Office 2013

- [ ] Установить пакет **MS Office 2013** на `nb-win` (обязательно компоненты **Access** и **Excel**).
- [ ] Проверить успешный запуск **Excel** (создание/открытие простой книги).
- [ ] Проверить успешный запуск **Access** (создание/открытие простой базы).

### 0.2 Подготовка действующих файлов Access и Excel

- [ ] Скопировать на `nb-win` **актуальные файлы Access/Excel**, в которых уже реализован целевой функционал ревизий и импорта (рабочая система).
- [ ] Зафиксировать пути к файлам (примеры, уточнить по факту):
  - [ ] `C:\FEMSQ-Legacy\ra_audits.accdb`
  - [ ] `C:\FEMSQ-Legacy\templates\*.xlsx`
- [ ] Убедиться, что файлы **открываются без ошибок** в Office 2013.

### 0.3 Подключение Access к MS SQL Server в Docker

- [ ] Убедиться, что контейнер `femsq-mssql` с базой **FishEye** запущен и доступен:
  - [ ] Проверить статус контейнера (Docker Desktop / `docker ps`).
  - [ ] Проверить доступ с `nb-win` по `localhost, 1433`.
- [ ] Настроить подключение **Access → MS SQL Server (Docker)**:
  - [ ] Создать/настроить необходимый **ODBC‑источник** или прямое подключение к базе `FishEye`, схема `ags`.
  - [ ] Открыть действующий Access‑файл и убедиться, что связка с таблицами/представлениями `ags` работает.
- [ ] Проверить базовый сценарий работы:
  - [ ] Открыть форму/модуль ревизий.
  - [ ] Убедиться, что данные читаются из базы **FishEye** в контейнере (без ошибок подключения).
