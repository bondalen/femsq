**Варианты развёртывания (выполнены в рамках плана):**

| Вариант | Каталог | Артефакты | Запуск |
|--------|---------|-----------|--------|
| **Fat JAR** | `26-0305` | Один JAR (~48–53 МБ), библиотеки внутри | `java -jar femsq-web-<version>.jar` |
| **Thin JAR + lib/** | `26-0309` | Thin JAR (~800 КБ) + каталог `lib/` (78 JAR, ~53 МБ) | `java -cp "femsq-web-<version>-thin.jar:lib/*" org.springframework.boot.loader.launch.JarLauncher` |

В Spring Boot 3.x `-Dloader.path=lib` не подхватывает внешние библиотеки; для thin JAR используется запуск через classpath и `JarLauncher` (см. `code/scripts/run-with-external-libs.sh`).

**Исполнено:** (1) **Fat JAR:** при первой попытке запуск падал с `NoClassDefFoundError: org/slf4j/LoggerFactory` — fat JAR повреждался при перезаписи в `LibManifestGenerator`. Исправлено: манифест добавляется через `jar uf` вместо полной перезаписи JAR. Выполнены пересборка, копирование JAR в `26-0305` и запуск в фоне; **http://localhost:8080/** — HTTP 200. (2) **Thin JAR + lib/:** сборка fat+thin через `build-thin-jar.sh` (версия 0.1.0.85), извлечение библиотек в `26-0309/lib` (`extract-libs-from-fatjar.sh`), копирование thin JAR в `26-0309`. Запуск через `-cp` и JarLauncher; проверка библиотек падала из‑за неверной версии в манифесте (только «SNAPSHOT»). В `LibManifestGenerator` исправлено извлечение версии из имени JAR (полная версия, напр. `0.1.0.85-SNAPSHOT`). Thin JAR пересобран, развёрнут в `26-0309`; Library validation passed, приложение запущено, HTTP 200.
