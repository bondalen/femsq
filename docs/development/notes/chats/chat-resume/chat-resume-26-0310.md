**Тема:** Подготовка среды для разработки импорта из Excel на nb-win: миграция с VMware на Hyper-V, Office 2013 в ВМ, подключение Access к тестовой БД FishEye в Docker.

## Связанные планы

## Контекст

- **Проблема:** после установки VMware Workstation Cursor AI на хосте nb-win переставал подключаться к провайдеру; связь восстанавливалась только после удаления VMware (сетевые адаптеры VMnet блокировали подключение).
- **Окружение:** ноутбук **nb-win** (Windows 11), WSL2 Ubuntu, Docker Desktop; тестовая БД **FishEye** в контейнере **femsq-mssql** (порт 1433).
- **Цель:** подготовить среду для разработки импорта из Excel на основе действующей Access-системы **без VMware** — запустить Office 2013 (Access + Excel) и подключить Access к FishEye в Docker.

## Выполненные задачи

### 1. Миграция с VMware на Hyper-V (Вариант A)

**Задача:** Заменить VMware на Hyper-V, чтобы Cursor и сеть работали стабильно, сохранив возможность запуска гостевой Windows с Office и Access.

**Шаги и решения:**
- Включены компоненты Windows: **WSL**, **Virtual Machine Platform**, **Microsoft-Hyper-V-All**, **HypervisorPlatform** (при включении HypervisorPlatform была временная ошибка Docker WSL integration; после `wsl --shutdown` и перезапуска Docker всё восстановилось).
- Выполнен бэкап папки ВМ: `D:\Program_Files\vm\vmware\w10-26-0310_BACKUP` (robocopy); удалены файлы suspend (`*.vmss`, `*.vmem`) из исходной папки.
- Конвертация диска VMDK → VHDX через **qemu-img** в WSL:
  - использованы однострочные команды и `wsl --%` для передачи аргументов в WSL (многострочные команды в PowerShell ломались из‑за кавычек и символов);
  - результат: `D:\Program_Files\vm\hyper-v\w10-26-0310\disk0.vhdx` (virtual size 60 GiB, disk size ~14.7 GiB).
- Создана ВМ Hyper-V **w10-26-0310** (Generation 2, UEFI), подключена к **Default Switch**; Secure Boot отключён.
- В гостевой ОС: вход по PIN; службы VMware Tools переведены в **Отключена** (полное удаление MSI не выполнялось).
- **Управление гостевой ВМ:** Enhanced Session Hyper-V отключён из‑за проблем с мышь/вводом; основная работа — по **RDP** (`mstsc`), буфер обмена хост↔гость работает.

**Результат:** ВМ Windows 10 в Hyper-V запускается и доступна с хоста nb-win без конфликта с Cursor и Docker.

### 2. Office 2013 и действующие файлы Access/Excel в ВМ

**Задача:** Установить Office 2013 (Access + Excel) в гостевой ВМ и разместить рабочие файлы действующей системы.

**Шаги:**
- В гостевой ВМ установлен и активирован **Microsoft Office профессиональный плюс 2013** (Access, Excel).
- Файлы Access/Excel извлечены из ZIP через **7-Zip** (проводник Windows давал ошибку 0x80004005 без запроса пароля).
- Пути зафиксированы (примеры): `C:\FEMSQ-Legacy\ra_audits.accdb`, `C:\FEMSQ-Legacy\templates\*.xlsx`.
- Проверена комплексная работа Access и Excel без подключения к SQL и с подключением; файлы открываются без ошибок.

**Результат:** Действующие файлы Access/Excel доступны в ВМ w10-26-0310 и открываются в Office 2013.

### 3. Подключение Access к FishEye в Docker (ODBC Driver 17)

**Задача:** Настроить связь Access с тестовой БД FishEye (контейнер femsq-mssql на хосте) так, чтобы таблицы и формы открывались без запроса пароля.

**Шаги и решения:**
- В гостевой ВМ создан пользовательский **ODBC DSN** к базе **FishEye** на хосте **nb-win,1433** (не localhost — SQL Server в Docker на хосте).
- Изначально использовался драйвер «SQL Server»; пароль в связях не сохранялся, при открытии появлялось окно входа.
- После перехода на **ODBC Driver 17 for SQL Server** при ручной привязке таблиц появился флажок «Сохранить пароль» — связи с этой галочкой открываются без запроса пароля.
- Программная установка `Connect` с UID/PWD и `dbAttachSavePWD` не привела к сохранению пароля после перезапуска Access; помогло только **пересоздание связей через UI** с галочкой «Сохранить пароль». Для массового пересоздания использованы процедуры: `RenameFishEyeLinksToOld`, ручная привязка через ODBC (Driver 17) с «Сохранить пароль», затем `DeleteOldFishEyeLinks`.
- Chrome в ВМ установлен через PowerShell (MSI `googlechromestandaloneenterprise64.msi`), т.к. старый URL давал 404.

**Результат:** Связанные таблицы и формы в Access отображают данные из FishEye (схема ags и др.); открытие файла без запроса пароля.

## Ключевые решения и особенности

| Вопрос | Решение |
|--------|--------|
| Конфликт VMware и Cursor | VMware удалена; используется Hyper-V (Вариант A). |
