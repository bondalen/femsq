**Назначение:** единый порядок сбора информации о таблицах, запросах и связанных объектах Access, чтобы переносить её в репозиторий (`docs/project/proposals/vba-analysis/`) без потери кириллицы и с полнотой, достаточной для Java/SQL Server.

## 1. Что передавать ассистенту (рекомендуемый минимум)

| Артефакт | Обязательно? | Зачем |
|----------|--------------|--------|
| **UTF-8-файл дампа** таблицы (из модуля `DumpTableDef_Extended` с `utf8FilePath`) | **Да**, для таблиц | Кириллица в `Description` / `Caption`, типы DAO, индексы, связи, `Format`; без mojibake |
| **Скриншот конструктора** таблицы (сетка полей + при возможности свойства) | Желательно | Быстрая сверка «как в Access» для человека |
| **Копипаст из Immediate** | Необязательно / **не для кириллицы** | Удобен для структуры без русского текста; кириллица часто превращается в `РџР»Р°РЅ…` |
| **Текст SQL запроса** | Для запросов — **да** | См. раздел 3: исходник удобно прислать как **`.txt`**, в репозитории хранится **`.access.sql`** |

**Итого:** для произвольной таблицы **оптимально** — **UTF-8 дамп + по желанию скриншот**. Вывод Immediate **не** заменяет файл, если важны русские подписи.

## 2. Таблицы (TableDef)

### 2.1. Правило «один файл на таблицу»

В каталоге [`access-queries/`](./access-queries/) для каждой описанной таблицы создаётся **ровно один** Markdown-файл:

**`{ИмяТаблицы}.table.md`**

В нём совмещаются:

- назначение таблицы, ссылки на SQL/staging и смежную аналитику;
- таблица полей (DAO-типы, NULL/REQ);
- **Caption / Description** по полям (из UTF-8-дампа или конструктора);
- индексы, связи (если есть в дампе);
- подводные камни (Immediate, бинарные `GUID`/`NameMap`, Err 3032 и т.д.);
- опционально — указание пути к `{ИмяТаблицы}.dump.utf8.txt`, если полный дамп закоммичен.

Отдельные файлы вида `{table}-field-captions.ru.md` **не заводим** — это дублирует один источник правды.

### 2.2. Инструмент

Модуль [`access-queries/DumpTableDef_RaImpNew.bas`](./access-queries/DumpTableDef_RaImpNew.bas): процедура **`DumpTableDef_Extended`** (универсальна для любого имени таблицы).

- Имя таблицы и путь к файлу — **строки в кавычках** (иначе Err 424).
- Пример:  
  `DumpTableDef_Extended "ra_ImpNew", True, False, True, "C:\temp\ra_ImpNew_dump.txt"`

### 2.3. Опционально в git

**`{ИмяТаблицы}.dump.utf8.txt`** — полный вывод дампа (копия с машины, где запускали Access).

### 2.4. Выполненный пример

| Элемент | Путь в проекте |
|---------|----------------|
| Таблица (всё в одном файле) | [`access-queries/ra_ImpNew.table.md`](./access-queries/ra_ImpNew.table.md) |
| Модуль DAO-дампа | [`access-queries/DumpTableDef_RaImpNew.bas`](./access-queries/DumpTableDef_RaImpNew.bas) |

Особенности, уже отражённые в документации:

- бинарные `GUID` / `NameMap` — в дампе `<binary/skipped>`;
- `ConflictTable` / `ReplicaFilter` — пропуск для локальных таблиц.

## 3. Запросы (QueryDef)

### 3.1. Порядок работы с ассистентом

1. Вы снимаете текст SQL в Access (конструктор → SQL или `Debug.Print CurrentDb.QueryDefs("Имя").SQL`).
2. Передаёте ассистенту как удобно — чаще всего **файл или вставка в чат в виде `.txt`** (сырой текст без специального оформления).
3. Ассистент **приводит** текст к виду, принятому в репозитории: один файл **`{ИмяЗапроса}.access.sql`** (форматирование, комментарий с именем запроса Access, при необходимости лёгкая правка переносов/точек с запятой под читаемость diff).

В репозитории хранится **только** `{ИмяЗапроса}.access.sql`; исходный `.txt` в проект не обязателен.

### 3.2. Примеры в репозитории

- [`access-queries/ra_ImpNewQuRa.access.sql`](./access-queries/ra_ImpNewQuRa.access.sql)  
- [`access-queries/ra_ImpNewQuRc.access.sql`](./access-queries/ra_ImpNewQuRc.access.sql)

Краткое назначение запроса фиксируется в [`ra-audit-btnAuditRun-analysis.md`](../../../development/notes/analysis/ra-audit-btnAuditRun-analysis.md) и/или одной строкой в начале `.access.sql`.

## 6. Чеклист «таблица описана достаточно»

- [ ] Есть **один** `{Имя}.table.md` с типами полей и NULL/REQ.
- [ ] В том же файле — **Caption/Description** (или явная отсылка к закоммиченному `.dump.utf8.txt`).
- [ ] Кириллица проверена по **UTF-8** дампу или скриншоту, не по копипасту Immediate.
- [ ] Указаны индексы и (если есть) связи из дампа.
- [ ] Зафиксировано назначение таблицы и соответствие объекту в SQL Server (если применимо).
- [ ] Для участвующих в сверке запросов — есть **`{Имя}.access.sql`** (после доработки из исходного `.txt`).
